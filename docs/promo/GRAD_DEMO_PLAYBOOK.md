# Paper_Rec 宣传演示：熟练研究生一周工作流

> 角色：研二 NLP/CV 方向研究生 · 已把 Paper_Rec 接到 Cursor  
> 场景：开题后推进「多模态对齐」主线，从检索到实验验收再到可投稿草稿  
> 用途：截图分镜 · 口播稿 · 录屏脚本（约 90–120 秒）

---

## 一句话卖点

**本地 Markdown 真源 + 研究主线 + 实验硬闸 + 诚信写作**——不是又一个「帮你写论文」的聊天框，而是跟课题同生共死的工作台。

---

## 研究生真实日程（宣传流程骨架）

| 阶段 | 研究生在干什么 | 产品触点 | 口播一句 |
|:---|:---|:---|:---|
| 0. 开场 | 打开 Wiki 总览 | 首页 brand + 三大模块 | 「我的文献、主线、实验都在本地同一处。」 |
| 1. 找论文 | Cursor 里 `/query_chinese` | Skill 对话（可切画面） | 「先按主线关键词多源检索，报告直接入库。」 |
| 2. 入库浏览 | 论文库筛选 status | `/pages` | 「每周入库一眼扫完，reading / done 不靠记忆。」 |
| 3. 双栏研读 | 打开一篇，摘录挂主线 | `/page/...` 双栏 | 「左边正文，右边挂 claim——摘录就是证据。」 |
| 4. 主线推进 | Thread 看 gaps / delta | `/threads/<id>` | 「假设、缺口、证据覆盖率，组会前 3 分钟复盘。」 |
| 5. 跑实验 | `/exp_loop` 或实验看板 | `/experiments` | 「指标对 target，不过硬闸就不谈投稿。」 |
| 6. 诚信写作 | draft + number-verify | Thread draft / Skills | 「数字和引用过闸，再谈导出 LaTeX。」 |
| 7. 收束 | 回到总览 / Skills | 首页或 Skills | 「Git 可回滚，导师要溯源随时打开。」 |

## 90 秒口播稿（可直接念）

> （0–8s · 首页）  
> 我是研二，课题是多模态对齐。文献散、实验散、草稿对不上数——后来把整条链路收进 Paper_Rec。

> （8–22s · 论文库）  
> 检索用 `/query_*` 多源打分入库；Wiki 论文库按状态筛，本周新进一眼看到。

> （22–42s · 双栏研读）  
> 读论文是双栏：左边原文，右边工具。选中一段，挂到研究主线的 claim 上，证据级别和置信度当场写清。

> （42–62s · 主线）  
> 主线页看覆盖率、开放问题和 Delta 推荐——不是收藏夹，是可推进的研究状态机。

> （62–78s · 实验）  
> 实验看板盯 target_score；lab / exp 过硬闸，数字对不上就不进草稿。

> （78–95s · 收束）  
> 草稿包 + 诚信闸门后再导出。数据在本地 Git，组会、开题、投稿同一套真源。  
> Paper_Rec——研究生的本地学术工作台。
